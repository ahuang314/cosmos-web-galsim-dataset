# Overview

This is the documentation page for the latest version of the dataset uploaded [here](https://zenodo.org/records/19188494) on Zenodo. 

This dataset, meant for use with SLSim, is a downselected version of the full COSMOS Web dataset, of which the master catalog `COSMOSWeb_mastercatalog_v1.fits` can be downloaded [here](https://cosmos2025.iap.fr/catalog.html), and the corresponding NIRCam images [here](https://cosmos2025.iap.fr/nircam.html#downloads).

The notebook used to generate this downselected catalog from the full dataset is provided in this repository.

# Cuts Applied to the Catalog

The downselected catalog was generated by applying the following cuts to the master catalog:

- sersic radius along the major axis > 0.15 arcseconds (5 pixels)
- magnitude < 21 in all of the following bands: F115W, F150W, F277W, F444W
- warn_flag = 0 (to ensure high quality sources; see documentation about quality flags [here](https://cosmos2025.iap.fr/catalog.html#quality-flags))
- redshift != -99 (some galaxies have their redshifts in the catalog set to -99; the documentation does not explain why)
- type = 0 (to select galaxies only; type = 1 is stars and type = 2 is QSOs)
- sources with contaminants that are within a distance D are excluded, where D is four times a given source's sersic radius
- sources with an image containing any zero-value pixels are excluded, filtering out sources that contain data reduction artifacts or are on the edge of the detector

# Description of the dataset

The downselected catalog is stored in the file named `COSMOSWeb_galaxy_catalog.fits`. This file contains an astropy Table with the following data:

- id: The identification number of the source
- sersic_radius: The fitted Sersic radius along the major axis, in arcseconds
- sersic_index: The fitted Sersic index
- sersic_angle: The fitted Sersic angle in radians, measured from East to North
- axis_ratio: The fitted axis ratio
- z: redshift
- sersic_fit_chi2: chi2 of the Sersic fit

The images corresponding to each galaxy are stored in a file named `COSMOSWeb_galaxy_{id}_image.fits`, where {id} is the id of the source. Inside this file is an empty PrimaryHDU followed by one ImageHDU for each band F115W, F150W, F277W, and F444W containing a cutout of the source. The size of each cutout is five times the sersic radius of the source.

# Version Release Notes

## Changes in Version 2

- Added a minimum size cut
- Changed the criteria for sources with contaminants to be excluded (used to be all sources with contaminants within a distance of 85 pixels were excluded)
- The downselected catalog now contains the "sersic_radius" column (used to be "angular_size", which was the geometric mean of the sersic radii along the major and minor axes)
- Changed image cutout sizes to be five times the sersic radius of the source (used to be 121x121 cutouts for all sources)
- Fixed a bug causing the cutouts for different bands to be misaligned
